Parsing Rules

In order to identify data in the collection software, a set of instructions must be configured for each file format uploaded into the system. The system uses these instructions, called Parsing Rules, to identify where data is located in a spreadsheet, and what type of data it is. Parsing rules must be configured separately for Trial Balances and Budgets.

It is possible for one property to require multiple parsing configurations.

Configure Parsing Rules

  1. Navigate to Company Info > Configure Parsing Rules
  2. Select Management Company from drop-down list
  3. Click +Parsing Configuration
  4. Name parsing configuration and click Add Parsing Configuration
  5. Select file format
  6. Select Location of Source Property ID/Name
    • Tab Name
    • In Cell
    • User Input
  7. Enter cell information from spreadsheet:
    • Source Account ID Column(s) – column(s) that contain the source account number and source account description
      • Remove – number of leading characters to remove from the source account number, if any
    • Closing Balance 1 – primary closing balance of a trial balance file
    • Closing Balance 2 – secondary closing balance of a trial balance file, if applicable
      • Some management companies report two closing balance columns, a debit column and a credit column. If trial balance does not list credits as negative (-), ‘Import as Negative’ must be checked
    • Opening Balance –opening balance column in the trial balance. Used for the opening balance validation.
    • Debit – debit column in the trial balance, if any. May be used to map cash flows.
    • Credit – credit column in the trial balance, if any. May be used to map cash flows.
    • Adjusted Debit – adjusted debit column in trial balance, if any. Used for the adjusted debit/credit validation.
    • Adjusted Credit – adjusted credit column in trial balance, if any. Used for the adjusted debit/credit validation.
    • Monthly/YTD – indicates whether debit and credit columns reflect month-to-date or year-to-date balances. Allows system to apply appropriate reversal rules to the data.

    8. Note: Configuration Name, File Format, Location of Source Property ID/Name, Source Account ID Column(s), and Closing Balance 1 are required regardless of file format.
