Properties

When a new property is added to the data collection service, it must be entered into the software. To do so, a property must already exist in Fusion, and Fusion access is required to locate many of the attributes needed to create the property's entry in the system.

Creating Properties

Create a Property

  1. Navigate to Client Info
  2. Click +Add Client Property
  3. Enter required property information:
    4. Client Property ID - Found in page location of Fusion

    Note: Client Property ID will be different than OP ID

    Client Property Name - Property Name in Fusion

    Navigate to Property > Details

    Client Partnership ID - OP ID in Fusion

    Navigate to OP Details > Details

    Client Partnership Name - OP Name in Fusion

    Navigate to OP Details > Details

    Address Information - Property Information in Fusion

    Navigate to Property > OP Details

    Investment Type - Fund Type in Fusion

    Navigate to OP Details > Details

    Acquisition Date - Found in Fusion

    Navigate to OP Details > Details

    Property Start Date - Lease Up date in Fusion

    Navigate to Monthly Reports > Occupancy

    Property End Date - Received by Client instruction
    See Client Reference Data Changes email
    Total Units - Total physical units in Fusion

    Navigate to Property > Unit Mix

    LIHTC Units - LIHTC units in Fusion

    Navigate to Property > Unit Mix

    Occupancy Delta - Used to add/subtract certain units from reported number
    • Manager unit
    • Non-revenue units
    Collection Rules - Based on client agreement, broken down by Trial Balance, Budgets, and Occupancy
    • Process - Data can be loaded and processed. No tracking on Tracking Grid. No email notifications.
    • Track, Process - Data can be loaded and processed, and is tracked on Tracking Grid. No email notifications.
    • Email, Track, Process - Data can be loaded and processed, is tracked on Tracking Grid, and receives email notifications.
    • Ignore - Property will be completely ignored.
  4. Click Save

Assigning Source Properties

Once a property has been added to the data collection software, the system must be told which spreadsheets are associated with the property. The data collection software must have a Source Property name to identify the appropriate properties from email file attachments sent by management companies.

When an email arrives that the system cannot automatically identify as belonging to property, a Source Property name must be assigned manually to the matching Client Property names. For example: file attachments named "Linden-Shawnee," "LINS," and "LINS trial_balance" must be manually assigned to "Linden's Apartments." After being assigned, the system will automatically identify those attachment names with the designated property.

Assign or Reassign Source Properties

  1. Navigate to Company Info > Assign Source Properties
  2. Select management company (or Unassigned) from Management Company drop-down list
  3. Locate Source Property and select matching property in Client Property drop-down list
  4. Click Save
